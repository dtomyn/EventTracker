# FR-013 Suggested Entry Connections

- Category: Functional
- Status: Baseline
- Scope: AI-assisted relationship discovery between timeline entries, storage of suggestions, and user acceptance/dismissal workflow.
- Primary Sources: `app/main.py`, `app/services/suggested_connections.py`, `app/templates/entry_detail.html`, `scripts/compute_suggested_connections.py`, `tests/test_entries.py`

## Requirement Statements

- FR-013-01 The system shall provide `scripts/compute_suggested_connections.py` to analyze entries in a timeline group and generate suggested relationship pairs.
- FR-013-02 The system shall use an AI provider to generate concise relationship phrases (3 to 8 words) describing how two entries might be related.
- FR-013-03 The system shall apply a distance threshold to filter candidate entry pairs based on semantic similarity, excluding pairs below the threshold.
- FR-013-04 The system shall limit the number of suggestions per entry to prevent excessive recommendations.
- FR-013-05 The system shall store generated suggestions in the `entry_suggestions` table with entry id, suggested entry id, relationship phrase, and status fields.
- FR-013-06 The system shall display pending suggestions on the entry detail page (`/entries/{entry_id}/view`) as clickable relationship cards.
- FR-013-07 The system shall provide `POST /api/suggestions/{suggestion_id}/accept` to accept a suggestion and create a bidirectional connection between entries.
- FR-013-08 The system shall provide `POST /api/suggestions/{suggestion_id}/dismiss` to dismiss a suggestion without creating a connection.
- FR-013-09 The system shall update suggestion status when accepted or dismissed and record the timestamp of the action.
- FR-013-10 The system shall gracefully handle provider failures during suggestion generation and allow the process to continue for remaining entries.
- FR-013-11 The system shall skip duplicate or reversed suggestions (A → B and B → A not both stored) to avoid redundancy.

## Acceptance Notes

- Suggestions are generated by running `uv run python -m scripts.compute_suggested_connections` on demand.
- The suggestion compute script iterates over all entries in a group and generates pairs based on semantic distance.
- Accepted suggestions create bidirectional entry connections stored in the `entry_connections` table.
- The entry detail page shows pending suggestions as distinct UI elements separate from saved connections.
- Suggestions do not block entry save or retrieval operations; they are purely optional enhancement.
